## 17:46-18:00 - Memory 目录混淆问题

**问题根源：** 系统中有两个 memory 目录

| 目录 | 路径 | 用途 |
|------|------|------|
| **系统 Memory** | `~/.openclaw/memory/` | 系统级数据（SQLite、向量索引） |
| **工作区 Memory** | `~/.openclaw/workspace/memory/` | 用户记忆文件（每日笔记、项目数据） |

**错误：** 我多次将文件写入错误的 `~/.openclaw/memory/`

**根本原因：** 依赖"记忆"而非"机制"，执行时没有停顿检查路径

**解决方案：**
1. 在技能文档中添加路径说明注释
2. 创建 `PATH-CHECKLIST.md` 作为写入前检查清单
3. 建立检查流程：停顿 3 秒 → 问路径 → 展开绝对路径 → 确认写入

**记忆口诀：** "工作区 = workspace，系统目录别乱写"
