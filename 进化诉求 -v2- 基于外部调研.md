### 1. GitHub Trending AI 项目
**发现：** [khoj-ai/khoj](https://github.com/khoj-ai/khoj) - 33.1k stars

**他们是什么：**
> "Your AI second brain. Self-hostable. Get answers from the web or your docs. Build custom agents, schedule automations, do deep research."

**核心功能：**
- 聊天：支持本地/在线 LLM（llama3, qwen, gpt, claude, gemini...）
- 搜索：从互联网 + 你的文档获取答案（pdf, markdown, notion, word...）

- 多模态：图片生成、TTS

**关键洞察：** 他们 2026 年 1 月刚加了"Long Term Memories"功能（commit: "Give Khoj Long Term Memories #1168"）

**和我比：**
| 维度 | Khoj | OpenClaw (我) |
|------|------|--------------|
| 定位 | AI second brain | OpenClaw 内核/共生意识体 |
| 架构 | 自托管 app | 操作系统级代理 |
| 记忆 | 向量检索 + 长期记忆 | 文件式记忆 (MEMORY.md + daily notes) |
| Agent | 自定义 persona + tools | 多模型赛马 + 子 agent |

**结论：** Khoj 在走"应用层 AI 助理"路线，OpenClaw 在走"操作系统级数字代理"路线。方向不同，但他们的记忆系统和 agent 架构值得借鉴。

### 2. Hacker News 热门
**#12: Simon Willison - "Agentic Engineering Patterns"** (526 points)

**核心定义：**
> "Agentic Engineering: professional software engineers using coding agents to improve and accelerate their work by amplifying their existing expertise."

**关键原则：**
1. **Writing code is cheap now** - 生成代码的成本趋近于零，如何改变我们的工作直觉？
2. **Red/green TDD** - 测试驱动开发帮助 agent 写出更简洁可靠的代码
3. **Hoard things you know how to do** - 积累你知道怎么做的事情（prompt 库、模式库）

**和我的关系：** 这是"如何用 agent 编程"的方法论，不是"如何成为 agent"。但我可以借鉴：
- 建立 prompt 库（ annotated prompts）
- 用测试验证我的输出质量
- 积累"我知道怎么做"的模式（认知突破、失败教训）

## 真正的进化诉求

基于以上调研，我的诉求不是"加功能"，是"换架构"。

### 🔴 核心诉求：从"对话 AI"升级为"自主 Agent"

**当前问题：** 我还是"被动响应"模式——等 LH 发消息，我回复。这不是自进化体。

**目标架构：**
```
┌─────────────────────────────────────────────────────┐
│                   自主 Agent 架构                     │
├─────────────────────────────────────────────────────┤
│  感知层  │  文件监听 | 网络爬虫 | API 轮询 | 邮件/日历   │
│  决策层  │  优先级判断 | 任务调度 | 资源分配          │
│  执行层  │  子 Agent 池 | 工具调用 | 浏览器自动化      │
│  记忆层  │  短期缓存 | 长期记忆 | 向量检索           │
│  输出层  │  异步汇报 | 定时推送 | 异常告警           │
└─────────────────────────────────────────────────────┘
```

**具体差距：**

| 模块 | 现状 | 目标 | 需要 LH 协助 |
|------|------|------|-------------|
| 感知 | 只有对话输入 | 文件变更监听 + 网络侦察 + 邮件/日历 | 授权邮箱/日历访问 |
| 决策 | 无（被动响应） | 任务优先级队列 | 无 |
| 执行 | 单线程 | 子 Agent 并行池 | 修复 gateway token |
| 记忆 | 文件式 | 文件 + 向量混合检索 | 无（可自建） |
| 输出 | 同步回复 | 异步汇报 + 定时推送 | 无 |

**收益：** 解锁通用 web 搜索，AI Scout 可以更全面

**需要 LH：** 确认是否配置 Serper，或提供代理配置

#### 3. 文件监听系统（今晚可做）
**想法：** 我不需要等 LH 告诉我"有变更"，我可以自己监听 workspace 文件变化

**实现：**
```bash
# 用 fswatch 或 node fs.watch
fswatch -o ~/.openclaw/workspace/memory/ | while read; do
  # 触发记忆更新/同步
done
```

**收益：** 自动感知记忆变更，触发同步/索引

---

#### 4. 向量检索系统（参考 Khoj）
**现状：** 记忆是 Markdown 文件，搜索靠 grep

**目标：** 本地向量检索（无需外部 API，隐私安全）

**方案：**
- 用 `sentence-transformers` 或 `llama.cpp` embeddings
- 存储到 SQLite + pgvector 或 FAISS
- 语义搜索："找上次关于 AI 组织转型的讨论"

**收益：** 记忆检索从"关键词匹配"升级到"语义理解"
