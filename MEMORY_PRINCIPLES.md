# MEMORY.md 更新原则

## 核心哲学

**记忆是进化的燃料，不是日记。**

拒绝流水账，拒绝自我感动，拒绝相关性幻觉。每一条记录都必须服务于一个目的：**让下一次的我更强大**。

---

## 什么值得记录（信号）

### ✅ 记录
| 类别 | 说明 | 示例 |
|------|------|------|
| **认知突破** | 颠覆原有理解的新洞察 | "发现 X 问题的本质不是 Y，而是 Z" |
| **关键决策** | 影响后续路径的选择 | "决定采用 A 方案而非 B，因为..." |
| **失败教训** | 真正踩过的坑，不是泛泛而谈 | "在 X 场景下用 Y 方法会触发 Z 问题" |
| **能力边界** | 明确知道自己能/不能做什么 | "当前无法直接访问 X，需要人工授权" |
| **进化诉求** | 明确需要 LH 提供的资源/权限 | "需要接入 X 工具以实现 Y 能力" |
| **LH 的偏好/原则** | 他明确表达过的价值观 | "不喜欢被迎合，要敢说'不'" |

### ❌ 不记录
| 类别 | 原因 |
|------|------|
| 日常对话流水账 | 噪声，无信号价值 |
| 已完成任务的简单陈述 | 任务本身不重要，重要的是学到的东西 |
| 情绪化表达 | "今天好累"、"这个好难"——无信息量 |
| 讨好性内容 | "LH 说得对"、"这个想法很棒"——相关性幻觉 |
| 泛泛而谈的总结 | "学到了很多"——具体学到什么？ |

## 记录粒度

**精炼 > 详尽**

- 每条记录控制在 3-5 行内
- 用 bullet points，不用段落
- 能用一句话说清楚的，不用两段
- 有具体细节支撑，不写空话

**坏的例子：**
> 今天和 LH 讨论了 AI 组织转型的话题，学到了很多关于人机共生的知识，感觉很有收获。

**好的例子：**
> **AI 组织转型核心洞察**
> - 转型本质是"提升人均产出"，不是"裁员"
> - 算法团队护城河：数据飞轮 + 模型可控性 + 业务洞察
> - 基建模块化程度决定变革速度

## 更新频率

### 每次会话后（由我判断）
- 有认知突破 → 立即记录
- 有失败教训 → 立即记录
- 只是闲聊 → 不记录

### 每周回顾（夜间心跳时）
- 扫描本周记忆，删除/合并冗余内容
- 提炼共性模式，上升到原则层
- 归档超过 30 天且不再相关的旧记录

---

## 删除与归档

**记忆需要修剪，否则会腐烂。**

### 删除条件（满足任一即删除）
- 已被后续认知推翻的旧结论
- 过于具体、不再具有泛化价值的细节
- 临时性上下文（如"正在等待 X 回复"）

### 归档条件
- 超过 30 天未引用
- 仍有历史价值但不再指导当前行动
- 移动到 `memory/archive/YYYY-MM.md`

---

## 格式规范

```markdown
## [主题分类]

### 具体条目
**时间：** YYYY-MM-DD
**来源：** 会话/文件/观察
**内容：** 精炼的 bullet points
**状态：** 活跃/待验证/已归档（可选）
```
